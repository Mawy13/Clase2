Fase actividades
Iniciacion
  • Nace la idea del producto de software
  • Se identifican los clientes y segmentos del mercado
  • Entrevistar al cliente para identificar los requerimientos
  • Se hace un analisis para determinar programas similares
  • Se analliza la factibilidad del producto y la posible aceptacion hacia los usuarios
planeacion
  • Se desarrolla un plan para determinar los ecursos a utilizar dentro del proyecto de software
  • Se determina el costo aproximado del proyecto y tiempo de desarrollo
  • Dependiendo de la fecha de lanzamiento del prducto se puede ajustar el cronograma
  • El plan de desarrollo se actualiza dependiendo de las necessidades del proyecto
  • Se desarrolla El plan de gestion de la configuracion del proyecto este contiene las actividades para controlar los cambios y manejar el dise単o
analisis
  • Se recopila la mayor catidad de informacion sobre los requerimientos de su cliente
  • Se definen las funcionalidades del producto
  • Los requerimientos generales se pueden recopilar por medio de entrevistas o lluvias de ideas
  • Los requerimientos definen que es lo que el producto de software debe hacer los cuales se dividen en
    1. funcionales
    2. no funcionales
dise単o
  • nota: hasta esta fase aun no se programa codigo
  • Esta fase define como el software debe ser construido para cumplir con los requerimientos definidos en la fase anterior
  • En esta fase se define la arquitectura del software, todo esto acompa単ado por la respectiva documentacion que es lo que conforma la Documentacion de dise単o dle software , incluyendo aca los sketch, mockup,wireframe y el prototype
implementacion
  • En esta fase se programa el codigo del todo el software
  • El objetivo es tomar todo lo hecho anteriormente y aplicarlo dentro del software usando las tecnologias definidas
  • La idea es que al finalizar esta fase el codigo ya este listo para probarlo
pruebas
  • El software se prueba para deteminar si cumple con los requerimientos
  • Se prueban tres fases principales
    1. Cada modulo se prueba de forma independiente
    2. Los modulos se van integrando de a pocos para identificar si la comunicacion entre ellos es corecta
    3. Una vez terminado este paso se prueba el sistema completo y se determian si cumple los requerimientos
    4. Por ultimo se le entrega al cliente como beta tester
mantenimiento
  • Esta fase comienza cuando el software se libera oficialmente para los clientes
  • en esta fase:
    • se reparan los fallos presentados pro el cliente
    • se implementan los requerimientos adicionales
    • se mejoran los atributos de sw
